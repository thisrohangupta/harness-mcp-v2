const app = createMcpExpressApp({ host });

**3. Add session lifecycle calls:**

In `onsessioninitialized` callback (inside `new StreamableHTTPServerTransport({...})`), add `sessionConnected()` AFTER `sessions.set(id, ...)`:

```typescript
onsessioninitialized: (id) => {
  const authContext = req.authContext;
  sessions.set(id, {
    server: server!,
    transport: transport!,
    lastActivity: Date.now(),
    authContext,
  });
  sessionConnected();
  log.info("Session created", { ... });
},
```

In `destroySession()` function, add `sessionDisconnected()` AFTER `sessions.delete(sessionId)`:

```typescript
function destroySession(sessionId: string): void {
  const session = sessions.get(sessionId);
  if (!session) return;
  sessions.delete(sessionId);
  sessionDisconnected();
  session.transport.close().catch(() => {});
  session.server.close().catch(() => {});
  log.info("Session destroyed", { sessionId, remaining: sessions.size });
}
```

Note: `sessionDisconnected()` is placed after `sessions.delete()` but before transport/server close — this ensures the gauge decrements even if close() throws. The `sessionConnected`/`sessionDisconnected` functions themselves never throw (gauge.inc/dec are synchronous and infallible in prom-client).
