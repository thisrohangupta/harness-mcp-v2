    The challenge: each `registerXxxTool()` function calls `server.registerTool()` internally, so we cannot wrap the handler externally without changing the individual tool files.

    **Approach:** Use the MCP SDK's server to intercept tool handlers. The `McpServer.registerTool()` method accepts `(name, config, handler)`. We need to wrap the handler BEFORE it reaches `server.registerTool()`.

    **Solution:** Create a proxy server wrapper that intercepts `registerTool` calls and wraps each handler with `withMetrics`:

    1. Add import at the top of `src/tools/index.ts`:
       ```typescript
       import { withMetrics } from "../metrics/tool-metrics.js";
       ```

    2. Create a helper function that wraps the McpServer's registerTool method:
       ```typescript
       function createInstrumentedServer(server: McpServer, harnessRegistry: Registry): McpServer {
         const originalRegisterTool = server.registerTool.bind(server);
         const instrumentedServer = Object.create(server) as McpServer;
         instrumentedServer.registerTool = function(name: string, ...rest: unknown[]) {
           // The handler is always the last argument
           const args = [...rest];
           const handler = args[args.length - 1];
           if (typeof handler === "function") {
             const wrap = withMetrics(name, harnessRegistry);
             args[args.length - 1] = wrap(handler as any);
           }
           return originalRegisterTool.call(server, name, ...args as any);
         } as typeof server.registerTool;
         return instrumentedServer;
       }
       ```

    3. In `registerAllTools()`, create the instrumented server and pass it to all register functions:
       ```typescript
       export function registerAllTools(server: McpServer, registry: Registry, client: HarnessClient, config: Config): void {
         const instrumented = createInstrumentedServer(server, registry);
         registerListTool(instrumented, registry, client);
         registerGetTool(instrumented, registry, client);
         registerCreateTool(instrumented, registry, client);
         registerUpdateTool(instrumented, registry, client);
         registerDeleteTool(instrumented, registry, client);
         registerExecuteTool(instrumented, registry, client);
         registerDiagnoseTool(instrumented, registry, client, config);
         registerSearchTool(instrumented, registry, client);
         registerDescribeTool(instrumented, registry);
         registerStatusTool(instrumented, registry, client, config);
         registerAskTool(instrumented, registry, client, config);
       }
       ```

    This approach modifies ONLY `src/tools/index.ts`. No individual tool files change. The proxy intercepts every `server.registerTool()` call and wraps the handler argument with `withMetrics(toolName, harnessRegistry)` before forwarding to the real server.

    **Important:** The McpServer.registerTool signature is `registerTool(name, config, handler)` where handler is the last positional arg. The proxy must handle this correctly regardless of the SDK's overloaded signatures.

    **Alternative simpler approach if the proxy pattern is too fragile:** Instead of a proxy, modify each `registerXxxTool` function signature to accept an optional `wrapHandler` parameter. But this changes 11 files — prefer the proxy approach first. If the proxy doesn't work due to McpServer internal checks, fall back to directly patching `server.registerTool` as a method override on the instance.
